The BBB is a multipurpose user programmable input device and also serves as an exercise product to hone my skills and try out different circuit solutions. Not all features and component make sense from a real product perspective, but serves as a learning experience.



It all started with the desire to have a more professional feel to my hobby and learning projects. So instead of using the classical tactile buttons forced down into a breadboard entangled in jumper wires, I bought some random mechanical keyboard switches and 3d-printed an enclosure for them. Then came the time to make the circuit board to empower this.

Feature creep. That is the spirit of this project. Because of personal projects I need the BBB need to be Zigbee enabled and preferably also wifi capable. Since I had already used the ESP32-C6 for such purposes I decided to use this.

\subsection{Power requirements}

Before any parts of the circuit can be properly designed, we need to determine the power and current requirements throughout the system.

The maximum peak current consumption of the ESP32-C6 is during WiFi transmission, which can be up to \qty{354}{\milli\ampere} at \qty{3.3}{\volt} which is about \qty{1.2}{\watt} \cite[66]{esp32-c6-datasheet}. The average power consumption will be much lower than this, but we need to design the power system to be able to handle this peak.

Each WS2812 2020 LED consumes about $3 \times \qty{12}{\milli\ampere} \approx \qty{40}{\milli\ampere}$ at full brightness. Which for the full matrix of 9 LEDs would be about \qty{360}{\milli\ampere}, which at \qty{5}{\volt} would be \qty{1.8}{\watt} \cite{ws2812-product-page}.

A speaker such as \url{https://www.electrokit.com/en/miniatyrhogtalare-8ohm-1w-11x15mm} driven with a MAX98357A class D amplifier can consume up to \qty{1}{\watt} at \qty{5}{\volt}, which is about \qty{200}{\milli\ampere}. It is possible to run it directly from $V_{SYS}$ from the battery but it cannot output the full power. And since the \qty{5}{\volt} boost regulator will be able to handle \qty{1.5}{\ampere} when fed with \qty{3.6}{\volt} from the battery I decide to put it on the \qty{5}{\volt} rail.

\subsubsection{Important features}
As soon as the input voltage or battery voltage is high enough, the device will power up. At power up, the Input Current Limit is updated to reflect the \sig{PSEL} pin (\qty{500}{\milli\ampere} or \qty{2.4}{\ampere}). When this is done, the system output \sig{} is soft-started and the switch converter for the battery charging is enabled.

If nothing is done, the device will automatically enter Default Mode which means it will autonomously act as a battery charger and charge the battery. To exit this mode, the host must write a 1 to the \sig{WD\_RST} but. Then the device enters Host Mode until the watchdog timer times out and \sig{WATCHDOG\_FAULT} is set and the device returns to Default Mode.

The default fast charging current \sig{ICHG} is \qty{2040}{\milli\ampere} which is way too much for the chosen battery. Setting the Input Current Limit \sig{IINDPM} using the pin \sig{PSEL} high limits this to \qty{500}{\milli\ampere}. This might still be too much since the proposed battery allows a maximum charging current of \qty{400}{\milli\ampere} and the step-down converter will allow a higher charging current than input current.

\decision{Will the charger have to be disabled somehow at startup, and later enabled by the host, to prevent over current to the battery? Can be handled via the $\bar{CE}$ pin.}[Yes, the current is way too large for this battery, the $\bar{CE}$ pin must be pulled up to prevent automatic charging.]

\decision{Will some sort of startup delay or under-voltage-lock-out be needed for the subsequent circuits so that the soft-start can work properly?}

The device can operate in boost-mode to convert the battery voltage to \qty{5}{\volt} and max \qty{1.2}{\ampere} for OTG use. This is controlled via I2C.

\subsubsection{Typical application}
The typical application is shown in \ref{fig:bq25601-typical-application}.

There are three explained design procedures for determining the Inductor, the input Capacitor and the output Capacitor respectively.

$I_{SAT} \ge I_{CHG} + \frac{1}{2} I_{RIPPLE}$

$D = \frac{V_{BAT}}{V_{VBUS}}$

\begin{equation}
    I_{RIPPLE} = \frac{V_{IN} \times D \times (1-D)}{f_s \times L} \iff L = \frac{V_{IN} \times D \times (1-D)}{f_S \times I_{RIPPLE}}
\end{equation}

The inductors saturation current must be larger than the maximum charge current plus half the ripple current. The larger the inductance, the lower the ripple current. All of the parameters except the inductance and ripple current is known. According to you neighbouring LLM the ripple current should be \qtyrange{25}{30}{\percent} of the maximum load current, so about \qty{100}{\milli\ampere}. Lets see where that lands us.

\begin{equation}
    D = \qty{3.2}{\volt} / \qty{5}{\volt} = 0.64
\end{equation}
\begin{equation}
    L = \frac{\qty{5}{\volt} \times 0.64 \times (1 - 0.64)}{\qty{1.5}{\mega\hertz} \times \qty{0.1}{\ampere}} =
    \frac{\qty{1.152}{\volt}}{\qty{0.15}{\mega\hertz\ampere}} = \qty{7.68}{\micro\henry} \approx
    \qty{6.8}{\micro\henry}
\end{equation}

The saturation current must be greater than $\qty{400}{\milli\ampere} + \qty{50}{\milli\ampere} = \qty{450}{\milli\ampere}$. The self resonance should be greater than $10 \times f_S$ the switching frequency, so more than \qty{15}{\mega\hertz}. For example the WIP322512A-6R8MLDG from INPAQ Technology Co., Ltd that comes in a standard 1210 package and has many second source alternatives.

The input capacitor should be able take care of the ripple current. Suggestions make me target two or three \qty{10}{\micro\farad} capacitors in 0805 package.
$I_{CIN}=I_{CHG} \times \sqrt{D \times (1 - D)}$

$I_{COUT} = \frac{I_{RIPPLE}}{2 \times \sqrt{3}} \approx 0.29 \times I_{RIPPLE}$

$\Delta V_O = \frac{V_{OUT}}{8LCfs^2} \left( 1 - \frac{V_{OUT}}{V_{IN}} \right)$
